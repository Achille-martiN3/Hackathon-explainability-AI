MODEL CARD : Nom du Modčle
1. Objectif du modčle
* Cas d'usage visé : (ex : classification d'images, analyse de sentiment, prédiction de prix...)
* Entrées : (ex : images 512×512, texte brut, tableau de variables...)
* Sorties : (ex : classe prédite, score de sentiment, valeur numérique...)

2. Données d'entraînement
* Dataset(s) utilisés : (liste + lien si open-source)
* Taille / diversité :

* Nombre total d'échantillons 
* Répartition des classes 
* Diversité (âge, contexte, scénarios, etc.)

* Limites connues : (déséquilibrage, manque de diversité, absence de certaines populations ou situations...)
3. Performances
* Métriques utilisées (accuracy, F1, BLEU...) :
* Résultats : (résultats globaux + éventuellement par sous-groupe pour l'équité)
 
4. Limites
* Risques d'erreur connus : (ex : mauvaise performance en faible luminosité, erreurs sur une catégorie rare...)
* Situations non couvertes : (données hors distribution, cas extręmes...)
* Risques de biais : (variables sensibles, déséquilibre des classes, population non représentée...)

5. Risques & mitigation
* Risques de mauvaise utilisation : (sur-interprétation des résultats, usage hors périmčtre, décisions critiques...)

* Contrôles mis en place : (seuils, validation humaine, messages d'avertissement, filtrage d'inputs...)

6. Énergie et frugalité
* Poids du modčle : (en Mo ou Go)
* Temps d'inférence : (sur CPU/GPU)
* Énergie estimée (CodeCarbon) : (ex : CodeCarbon, approximation kWh/CO2)

7. Cyber
* Sécurisation des entrées : (détection d'inputs malveillants, vérification format/valeurs, protections anti-prompt injection)
* Secrets protégés : (clés API sécurisées, variables d'environnement, jamais dans le repo Git)
